# AI Instructions

## Important: Always Read AI Documentation First

Before working on this project, **always** read all documentation in the `/AI` folder first. Start with `README.md` in the AI folder, then proceed to other files.

### Required Reading Order:
1. Read `AI/README.md` first (entry point)
2. Read remaining files in `/AI` directory
3. Follow any specific instructions found there
4. Only then proceed with development tasks

### Why This Matters:
- Contains project-specific AI guidelines
- Includes architectural decisions and patterns
- Provides context for code style and conventions
- May contain critical constraints or requirements

**Failure to read the AI documentation first may result in incorrect implementations, wasted time, or code that doesn't align with project standards.**

## Logging Requirements

After completing any programming task, AI must document work in `LOGS.md`:

- **Location**: `AI/LOGS.md`
- **Content**: Summary of changes made, files modified, and decisions taken
- **Format**: Timestamped entries with clear descriptions
- **Include**: Any issues encountered and how they were resolved

## Debugging Guidelines

For debugging steps and troubleshooting procedures, see `AI/DEBUG.md`.

## PTV API Reference

When implementing or modifying PTV API functionality, **always** read `AI/API_Documents.json` first. This file contains:

- OpenAPI/Swagger specification for the PTV Timetable API
- Available endpoints and their parameters
- Request/response schemas
- Authentication requirements

**Location**: `AI/API_Documents.json`

**When to read**: Before implementing any new PTV API features or modifying existing API calls.
